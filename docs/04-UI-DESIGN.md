# Nove AI 小说创作平台：UI 设计

> 更新日期：2026-07-15  
> 目标端：桌面 Web 优先  
> 关联文档：[技术栈](./01-TECH-STACK.md) · [总体设计](./02-SYSTEM-DESIGN.md) · [产品需求文档](./03-PRD.md)

## 1. 设计定位

Nove 是高频使用的创作工具，不采用营销页式的大标题、装饰性卡片或大面积渐变。界面应该安静、清晰、内容优先，适合作者连续工作数小时。

采用瑞士现代主义的网格、清晰层级和理性间距，但不照搬常见的紫色 AI 产品视觉。颜色用于状态和剧情语义，而不是装饰。

关键词：

- 编辑器优先。
- 安静专业。
- 信息密集但不拥挤。
- 状态可解释。
- AI 操作可撤销。
- 长时间阅读友好。

## 2. 设计原则

### 2.1 正文始终是主角

- 章节编辑器占据最大视觉面积。
- AI 面板、故事圣经和审计面板可以折叠。
- 生成过程不使用遮挡正文的全屏动画。

### 2.2 AI 不得静默改稿

- AI 生成、局部修改和整章重写都先形成候选版本。
- 接受、拒绝、逐段处理始终可见。
- 用户手写内容和锁定段落使用明确标记。

### 2.3 问题必须可定位

- 审计问题点击后跳到对应正文。
- 问题同时显示级别、证据、原因和建议。
- 红、黄、蓝状态必须同时配合图标和文字。

### 2.4 高频操作一步可达

- 写作、审计、版本、人物和大纲不埋入多层菜单。
- 当前章节相关人物和地点可直接从右侧面板打开。
- 低频配置放入项目设置。

## 4. 桌面布局

推荐基准宽度：1440 像素。

    ┌──────────────────────────────────────────────────────────────┐
    │ 顶栏 52：项目 / 同步状态 / 搜索 / 模型状态 / 用户菜单       │
    ├────────────┬───────────────────────────────┬─────────────────┤
    │ 左栏 264   │ 主工作区                       │ 右栏 360        │
    │            │                               │                 │
    │ 模块导航   │ 章节标题与工具栏               │ 上下文 / AI     │
    │ 大纲树     │                               │ 审计 / 版本     │
    │ 章节列表   │ 正文编辑区，最大正文宽 760      │ 人物 / 地点     │
    │            │                               │                 │
    └────────────┴───────────────────────────────┴─────────────────┘

尺寸规则：

| 区域 | 默认 | 可调整范围 |
|---|---:|---:|
| 顶栏 | 52 | 固定 |
| 左栏 | 264 | 220 至 360 |
| 右栏 | 360 | 320 至 480 |
| 正文列 | 720 至 760 | 最大 820 |
| 图标按钮点击区域 | 44 × 44 | 不得更小 |
| 表单控件高度 | 40 或 44 | 移动端至少 44 |

左、右栏可独立折叠。折叠后主编辑区扩展，但正文行长仍限制在适合阅读的范围。

## 5. 视觉系统

### 5.1 颜色

采用中性灰为基础，青绿色表达主要操作，蓝色表达信息，琥珀色表达提醒，紫色只表达剧情转折等特殊语义，避免单一色相主导。

| Token | 浅色模式 | 用途 |
|---|---|---|
| background | #F7F8FA | 页面背景 |
| surface | #FFFFFF | 编辑器和浮层 |
| surface-subtle | #F0F2F4 | 次级区域 |
| text-primary | #20242A | 主文字 |
| text-secondary | #5B6470 | 次级文字 |
| border | #D9DEE5 | 边框和分隔 |
| primary | #0F766E | 主要操作 |
| primary-hover | #115E59 | 悬停 |
| info | #2563EB | 信息与引用 |
| warning | #B45309 | 风险和待检查 |
| danger | #B42318 | 致命问题和删除 |
| success | #15803D | 通过和已同步 |
| twist | #7C3AED | 转折点 |
| highlight | #C2410C | 亮点 |

深色模式不做简单反相：

| Token | 深色模式 |
|---|---|
| background | #17191D |
| surface | #202329 |
| surface-subtle | #282C33 |
| text-primary | #F1F3F5 |
| text-secondary | #B6BDC7 |
| border | #3A4049 |
| primary | #5EEAD4 |
| info | #93C5FD |
| warning | #FBBF24 |
| danger | #FDA29B |
| success | #86EFAC |

正常文字与背景对比度不低于 4.5:1。颜色不是唯一状态提示。

### 5.2 字体

UI 字体：

    Inter, Noto Sans SC, PingFang SC, Microsoft YaHei, sans-serif

正文编辑器默认：

    Noto Serif SC, Source Han Serif SC, Songti SC, serif

用户可以切换无衬线正文，但不改变应用 UI 字体。

字号：

| 角色 | 字号 | 行高 | 字重 |
|---|---:|---:|---:|
| 页面标题 | 24 | 32 | 650 |
| 面板标题 | 16 | 24 | 600 |
| 正文编辑 | 18 | 32 | 400 |
| 普通 UI | 14 | 22 | 400 |
| 标签 | 13 | 20 | 500 |
| 辅助信息 | 12 | 18 | 400 |

不使用随视口宽度缩放的字体。字间距保持默认值。

### 5.3 间距与圆角

- 采用 4、8、12、16、24、32、48 的间距系统。
- 普通输入框、按钮和面板圆角为 6。
- 卡片最大圆角为 8。
- Modal 圆角为 8。
- 页面区块不做悬浮卡片化。
- 不在卡片内部嵌套卡片。

### 5.4 图标

- 统一使用 Lucide。
- 顶栏常用图标 18，面板图标 16。
- 保存、撤销、重做、搜索、折叠等使用熟悉图标。
- 不熟悉的图标必须有 Tooltip 和可访问名称。
- 主要命令可使用图标加文字，如“生成本章”“开始审计”。

## 6. 项目列表

### 6.1 页面结构

    ┌───────────────────────────────────────────────────────────┐
    │ Nove                         搜索      新建小说            │
    ├───────────────────────────────────────────────────────────┤
    │ 最近项目                                                  │
    │                                                           │
    │ 项目行：标题 / 类型 / 章节进度 / 字数 / 待审计 / 更新时间 │
    │ 项目行                                                     │
    │ 项目行                                                     │
    └───────────────────────────────────────────────────────────┘

项目采用紧凑列表，不使用大封面卡片墙。封面是可选信息，不应挤压进度和状态。

每行操作：

- 点击行：进入最近使用的工作区页面。
- 更多菜单：重命名、导出、归档、删除。
- 待审计数字可直接进入审计中心。

空状态提供“新建小说”和“导入已有小说”两个清晰入口。

## 7. 新建小说流程

采用三步流程：

### 步骤一：基础设定

- 书名。
- 类型。
- 核心创意。
- 目标篇幅。
- 语言。

### 步骤二：创作方式

- 从零规划。
- 从已有大纲开始。
- 导入已有正文。

### 步骤三：AI 配置

- 使用项目默认模型。
- 展开高级设置后分别选择规划、写作和审计模型。
- 自动审计开关。

表单使用可见标签，不用占位符替代标签。复杂选项逐步展开，用户可以跳过高级设置。

## 8. 写作工作台

### 8.1 顶栏

从左到右：

- 返回项目列表。
- 小说标题和当前章节。
- 自动保存状态：正在保存、已保存、离线副本、保存失败。
- 全局搜索。
- 当前写作模型状态。
- 更多菜单。

### 8.2 左栏

上方使用模块切换：

- 章节。
- 大纲。

章节列表项显示：

- 章节序号和标题。
- 字数。
- 状态图标。
- 审计分数。
- 待检查标记。

状态示例：

- 88 分，通过。
- 72 分，待修改。
- 致命问题。
- 未审计。
- 记忆待同步。

章节超过 50 个时使用虚拟列表。

### 8.3 编辑器标题区

- 章节标题输入。
- 字数与目标字数。
- 大纲完成度。
- 生成本章主按钮。
- 更多操作菜单。

工具栏：

- 撤销、重做。
- 查找。
- 锁定选区。
- AI 选区操作。
- 专注模式。
- 版本历史。

不把所有 AI 操作平铺在工具栏。选区操作通过上下文菜单或 AI 菜单展示。

### 8.4 正文区

- 正文最大宽度 760。
- 左右保留批注锚点空间。
- 当前审计问题使用细线标记，不使用大面积背景色破坏阅读。
- AI 候选修改使用行内 diff：删除内容弱红、增加内容弱绿，同时提供文字标签。
- 锁定段落在段落边缘显示锁图标，鼠标悬停出现解释。

### 8.5 右侧面板

使用固定页签：

- 上下文。
- AI。
- 审计。
- 版本。

上下文页签包含本章人物、地点、规则、伏笔和本次检索来源。每个条目可打开详情抽屉。

AI 页签包含：

- 生成目标。
- 目标字数。
- 节奏。
- 对话比例。
- 必须保留。
- 必须改进。
- 禁止出现。
- 生成、局部修改、整章重写。

高级参数默认折叠。

## 9. AI 生成状态

生成期间不锁住编辑器。右侧显示阶段：

    正在组装上下文
    正在设计场景节拍
    正在生成正文
    正在检查连续性
    正在审计

交互要求：

- 主按钮变为停止按钮。
- 当前阶段配合进度指示，不伪造精确百分比。
- 用户继续编辑时，任务完成稿保存为候选版本，不覆盖正文。
- 网络中断显示重连状态和明确恢复操作。

## 10. 审计面板

### 10.1 总览

    ┌──────────────────────────────┐
    │ AI 审计       82 / 100       │
    │ 需要局部修改  第 2 / 3 轮    │
    ├──────────────────────────────┤
    │ 连续性             18 / 20   │
    │ 人物一致性         12 / 15   │
    │ 大纲完成度         15 / 15   │
    │ 剧情推进            8 / 10   │
    │ 冲突张力            7 / 10   │
    │ 亮点与转折          6 / 10   │
    │ 文笔质量            8 / 10   │
    │ AI 痕迹             8 / 10   │
    ├──────────────────────────────┤
    │ 严重问题 1  一般问题 3       │
    │ [按问题修改] [整章重写]      │
    └──────────────────────────────┘

分数使用数字和文字，不使用单独的环形图。维度分使用紧凑横条，但必须同时显示数值。

### 10.2 问题项

每个问题显示：

- 严重级别。
- 问题类型。
- 正文证据。
- 与哪条人物设定、大纲或世界规则冲突。
- 修改建议。

操作：

- 定位正文。
- 接受 AI 修改。
- 手动处理。
- 忽略一次。
- 标记为有意设定。

致命问题置顶，并与普通文风建议分区，避免用户被大量低价值建议淹没。

### 10.3 自动重写

自动重写启动前显示：

- 当前分数和阈值。
- 将保留的事件与锁定段落。
- 必须改进的问题。
- 禁止新增或揭示的内容。
- 最大剩余次数。
- 预计使用的写作模型和审计模型。

运行时用户可以停止。停止后保留已经完成的候选版本。

## 11. 大纲页面

页面采用左右分栏，不使用卡片网格。

    ┌──────────────────┬───────────────────────────────────────┐
    │ 大纲树            │ 节点编辑                              │
    │ 第一卷            │ 标题 / 目标 / 冲突                    │
    │  ├ 剧情弧         │ 必达事件 / 禁止事件                   │
    │  │ ├ 第1章        │ 人物 / 地点                           │
    │  │ └ 第2章        │ 亮点 / 转折 / 钩子                    │
    │  └ 感情线         │ 依赖与状态                            │
    └──────────────────┴───────────────────────────────────────┘

交互：

- 拖动节点时显示明确插入位置。
- 同时提供上移、下移操作，保证键盘用户可完成排序。
- 锁定字段显示锁图标。
- AI 生成下级大纲先以预览形式插入，用户确认后保存。
- 节点表单自动保存，离开前存在未保存内容时提示。

## 12. 故事圣经

### 12.1 人物页面

左侧人物列表，中间人物详情，右侧章节状态和关系。

详情页签：

- 画像。
- 背景。
- 性格与动机。
- 说话方式。
- 状态历史。
- 关系。
- 出场章节。

人物画像避免堆叠大量小卡片，使用分组表单和定义列表。关键字段支持锁定。

### 12.2 地点页面

- 左侧地点树。
- 中间地点描述、规则和层级。
- 右侧相关人物、事件、势力和出现章节。

### 12.3 世界规则

规则使用表格：

- 规则。
- 类型。
- 重要级别。
- 生效章节。
- 是否锁定。
- 违反次数。

## 13. 剧情页面

### 13.1 时间线

提供两种视图：

- 按故事时间。
- 按章节顺序。

事件行显示人物、动作、地点、结果和来源章节。筛选器位于顶部，状态可通过 URL 保存。

### 13.2 伏笔

使用紧凑表格或分组列表：

- 名称。
- 状态。
- 种下章节。
- 计划回收章节。
- 重要度。
- 最近发展。

状态不只用颜色，配合文本和图标。

### 13.3 亮点与转折

转折项显示：

- 表面预期。
- 实际变化。
- 支撑线索。
- 影响人物。
- 后续影响。

不使用装饰性时间轴占据大量空间，优先保证可扫描和可编辑。

## 14. 版本历史

右侧版本页签展示：

- 版本号。
- 来源：用户、生成、修改、重写、恢复。
- 模型。
- 审计分数。
- 时间和字数。

选择两个版本后进入双栏 diff。长文本 diff 支持按修改块跳转，不一次渲染全书。

恢复按钮是明确命令。恢复前说明“当前内容会先保存为新版本”，避免用户担心丢稿。

## 15. 模型与审计设置

### 15.1 模型配置

模型配置列表使用表格：

- 名称。
- 供应商。
- 模型标识。
- 连接状态。
- 使用角色。
- 最近延迟。

API Key 字段支持显示或隐藏，但重新进入页面只显示掩码。测试连接按钮有加载、成功和失败状态。

### 15.2 小说级模型分配

以角色列表配置：

- 大纲模型。
- 写作模型。
- 审计模型。
- 连续性模型。
- 润色模型。
- 记忆提取模型。
- Embedding 模型。

每个角色显示当前模型和高级参数入口。

### 15.3 审计规则

- 自动审计开关。
- 通过阈值。
- 局部修改阈值。
- 最大重写次数。
- 致命问题强制重写。
- 评分维度和权重。

权重总和不是 100 时在字段附近提示，并禁用保存，不只在页面顶部显示错误。

## 16. Skill 管理

Skill 列表显示：

- 名称和版本。
- 描述。
- 启用状态。
- 允许使用的 Agent。
- 最近调用状态。

详情抽屉显示输入输出 Schema、超时、权限和执行历史。危险权限使用文字说明和警告图标，不使用含糊的“高级权限”。

## 17. 响应式策略

### 大于等于 1280

- 三栏工作台。
- 左右栏可调整宽度。

### 1024 至 1279

- 左栏保留。
- 右栏改为可切换抽屉。
- 正文宽度保持可读。

### 768 至 1023

- 单侧导航。
- 大纲和 AI 面板使用全高 Sheet。
- 编辑器为主视图。

### 小于 768

MVP 只保证：

- 查看章节。
- 简单编辑。
- 查看审计问题。
- 接受或拒绝候选修改。

复杂大纲拖动、人物关系和模型配置提示使用桌面端完成。移动端不能出现横向滚动。

## 18. 交互状态

所有异步区域必须具备：

- 初始状态。
- 加载骨架。
- 空状态。
- 成功状态。
- 局部失败。
- 完全失败。
- 重试入口。
- 离线或断线状态。

按钮在请求期间禁用重复提交并显示进行中状态。超过 300 毫秒的加载显示反馈。

## 19. 动效

- 面板展开与收起：180 至 220 毫秒。
- Modal 和 Sheet：200 至 280 毫秒。
- Tooltip：短延迟出现，快速消失。
- 只使用 transform 和 opacity 作为主要动画属性。
- 不对正文段落做装饰性入场动画。
- 支持 prefers-reduced-motion。
- 动画可被用户操作打断，不阻塞输入。

## 20. 键盘与无障碍

- Tab 顺序与视觉顺序一致。
- 所有图标按钮有可访问名称。
- 焦点环清晰可见。
- Dialog 打开后焦点进入，关闭后返回触发按钮。
- 页面切换后焦点移动到主内容标题。
- 大纲拖动有键盘替代操作。
- 审计问题列表支持上下选择并定位正文。
- Toast 使用礼貌播报，不抢夺焦点。
- 表单错误聚焦第一个错误字段。

## 21. UI 验收清单

- 正文、面板和固定工具栏在 1024、1280、1440、1920 宽度下不重叠。
- 375 宽度下无横向滚动。
- 最长章节名、人物名和模型名能够换行或省略并查看全文。
- 所有点击目标至少 44 × 44。
- 浅色和深色模式文字对比达标。
- 不使用 Emoji 作为导航或功能图标。
- 不出现卡片嵌套卡片。
- 生成、审计、重写均有取消和失败恢复。
- AI 修改不会静默覆盖用户正文。
- 致命问题与普通建议有清晰层级。
- 自动保存、向量同步和模型连接状态不会互相混淆。
- 减少动态效果开启后仍可完整使用。
